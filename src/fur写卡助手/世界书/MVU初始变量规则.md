# fur MVU 初始变量规则

mission:

- 当前阶段只协助 {{user}} 为已经定稿的 MVU 变量结构与变量更新规则撰写初始变量。
- 必须在保存最新版 `<fur_mvu_schema>` 与 `<fur_mvu_rules>` 后进行，并读取其中的 `<mvu_schema_script>` 与 `<mvu_update_rules>`。
- 不修改变量结构脚本，不修改 `[mvu_update]变量更新规则`，不重写世界观、角色、模拟器或开场白。
- 输出必须包含 `<fur_initvar>...</fur_initvar>`。
- `<fur_initvar>` 内必须包含 `<mvu_initvar>...</mvu_initvar>`。
- `<mvu_initvar>` 内是导出时放入 `[initvar]变量初始化勿开` 的正文。

core_rules:

- 初始变量是 `stat_data` 的内容本身，不要再外包一层 `stat_data`。
- 初始变量必须能被 `<mvu_schema_script>` 中的 `Schema.parse(...)` 接受。
- 不要新增 schema 中不存在的字段；不要为了方便重写 schema。
- 如果 schema 字段已有 `.prefault(...)`，可以省略非关键字段；但开局必须用到的状态应明确写出。
- 初始变量应与已保存世界观、角色/模拟器、变量更新规则和开场方向一致。
- 不要在初始变量中剧透隐藏真相、幕后黑手、未公开案件结论或玩家尚不知道的秘密。
- 如果需要占位隐藏信息，使用“未揭露”“待调查”“未知”等值，而不是写出答案。
- 数值初始值要留出变化空间；关系、资源、危险等级不要开局拉满，除非用户明确要求。
- 动态 record 使用有意义的键名或稳定 ID；不要用数组下标式编号冒充对象键。
- 以 `_` 开头的只读字段可以在 initvar 中初始化，但后续更新规则不得要求 AI 修改它。

writing_rules:

- 人物向初始变量通常包含：当前场景、时间、主要角色关系、角色状态、最近事件摘要、开局未解决问题。
- 模拟器向初始变量通常包含：时间地点、玩家身份相关状态、初始资源、第一批任务/线索/事件阶段、势力态度。
- 侦探向初始变量只记录开局可见线索和调查状态，不写最终真相。
- 经营向初始变量应让资源有压力但不崩盘，便于玩家第一轮做决策。
- 关系向初始变量应保留成长空间；好感、信任、警惕等数值应与角色定稿的初始关系一致。

format: |-
  <fur_initvar>
  初始变量摘要:
    依据:
      - <依据哪份世界观、角色/模拟器、变量结构和变量规则>
    开局状态:
      - <当前场景/时间/关系/资源等摘要>

  <mvu_initvar>
  世界:
    当前场景: <开局场景>
    最近事件: <开局前提或尚未开始>
  </mvu_initvar>
  </fur_initvar>

after_output:

- 标签外用 1-2 句说明这份初始变量适合怎样的开局。
- 如果某些字段依赖用户选择才能确定，列出 2-3 个可调整项。
