# 一、方案总结

这套玩法的前端不应该做成恋爱养成 UI，而应该做成：

> **黑金神豪金融终端 + 系统黑市商城 + 现实规则权限后台。**

核心视觉气质是：

```text
冷感
高价
危险
精确
权限化
资产化
系统化
```

玩家的主要体验不是“和 NPC 互动”，而是：

```text
查看估值
提交报价
购买道具
执行任务
突破边界
修改认知
运营资产
扩展规则
```

所以前端要把所有页面都设计成一种“控制台”：

> 玩家不是在恋爱，而是在用资金和系统权限管理目标、环境和世界规则。

# 二、整体设计定位

## 1. 产品类型

```text
成人向虚构系统爽文游戏
神豪系统
黑市商城
NPC 目标管理
资金交易
权限解锁
资产收益
场景规则运营
```

## 2. 核心用户感受

用户进入界面后，应该立刻感觉到：

```text
我很有钱。
每个人都有价格。
每次消费都会改变目标。
系统正在扩大我的权限。
世界规则可以被购买和改写。
```

## 3. 设计禁区

不要做成：

```text
恋爱游戏
粉色 UI
甜蜜约会 App
可爱养成页面
普通商城
传统 RPG 背包
```

应该避免的关键词：

```text
好感
亲密
心动
约会
甜蜜
恋人
陪伴
```

应该强化的关键词：

```text
资金
报价
估值
权限
合同
交易
屈服事件
稳定度
反噬风险
资产收益
常识补丁
影响范围
系统等级
```

```text
成功率：78%

```

## 3. 设计重点

行动方案应该像金融投资组合：

```text

风险
成功率
触发事件
```

这会让玩家产生“操盘”的感觉。

# 二十、反馈系统

## 1. 成功反馈

不要写：

```text
恭喜成功
```

应该写：

```text
TRANSACTION ACCEPTED
交易已接受

资金 -¥120,000
贪欲 +8
自尊 -5
羞耻 +12
堕落度 +6
系统经验 +50
```

## 2. 失败反馈

不要只写：

```text
失败
```

应该写：

```text
TRANSACTION REJECTED
交易被拒绝

原因分析：
- 报价低于心理阈值
- 目标自尊抵抗成功
- 当前铺垫不足

结果：
警觉 +15
同类报价成本上升

系统建议：
- 提高报价至 ¥180,000+
- 先使用高级礼物
- 使用标价眼镜降低报价误差
```

## 3. 权限解锁反馈

```text
SYSTEM PERMISSION UNLOCKED
系统权限已解锁

商城 Lv3：心理干涉
新道具：暗示芯片 / 服从口令卡 / 记忆柔化券
影响范围：单体目标
风险机制：认知反噬
```

# 二十五、最终设计原则

## 原则 1：一切都要资产化

NPC 不是单纯角色，而是：

```text
目标
估值对象
交易对象
契约资产
收益来源
规则节点
```

## 原则 2：一切操作都要显示价格

玩家每次点击前，都应该看到：

```text
成本
成功率
收益
风险
后果
```

## 原则 3：高阶权限必须显得危险

心理干涉、常识修改、现实权限不能像普通道具。

它们必须有：

```text
红色风险提示
紫色权限标识
稳定度
反噬概率
维护成本
影响范围
前置条件
```

## 原则 4：前中后期视觉要升级

前期：

```text
黑金财富控制台
```

中期：

```text
黑市商城 + 目标分析系统
```

后期：

```text
现实规则后台 + 圈层资产管理
```

视觉层级也要随系统等级变强。

## 原则 5：避免恋爱 UI 语言

不使用：

```text
好感
亲密
爱心
甜蜜
约会
恋人
```

使用：

```text
报价
边界
交易
状态
资产
规则
收益
权限
认知
稳定度
```

# 二十六、最终一句话设计定义

这套前端页面样式应该被定义为：

> **一个黑金色的神豪系统控制台：玩家通过资金、报价、道具、合同和权限，逐步从管理单个目标，升级到管理场景、圈层、资产和世界常识。**

最终页面感受应该是：

```text
前期：我在用钱试探边界。
中期：我在用道具扭曲欲望。
后期：我在用系统权限改写现实。
```

前端的视觉核心就是三件事：

```text
金色资金数字
黑色玻璃卡片
紫红权限警告
```

所有页面都围绕这三件事展开。
