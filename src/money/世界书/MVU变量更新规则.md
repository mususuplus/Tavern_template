# money MVU 变量更新规则

<money_mvu_rules>
变量规则摘要:
  更新原则:
    - 轻量 MVU 只维护“资金、EJS 路由、目标状态、已购买道具、任务、首次事件”。
    - 不再维护资产、常识规则明细、商城库存、经验、认知日志、社会价值等级等大变量。
    - 常识补丁、资产化、商城商品效果写进剧情与 `目标库.最近交易`，必要时影响 `系统.风险等级`、`目标库.当前状态`、`目标库.堕落度`、`目标库.警觉度`。
    - `界面状态` 是 EJS 控制变量，不能删除，也不要当作结算结果本身。
    - `目标库`、`商城.已购买道具`、`任务` 使用可扩展对象；新增单个条目时允许 `insert /目标库/目标名`、`insert /商城/已购买道具/道具名`、`insert /任务/任务名`。
    - 批量新增多个条目时，也可以 replace 整个父对象，并保留已有旧键。

变量更新规则:
  系统:
    等级:
      check:
        - 只在任务完成、商城权限升级或剧情明确系统升级时提高。
        - 等级范围 1-5。
        - 等级变化时同步检查 商城权限。
    当前资金:
      check:
        - 领取今日额度、任务奖励、剧情资金到账时增加。
        - 报价成交、购买道具、封口费、场地费、常识补丁成本明确发生时减少。
        - 被拒绝且未付款时不扣款。
    今日额度:
      check:
        - 可随等级提升而提高；Lv1 建议 500000，Lv2 1500000，Lv3 5000000，Lv4 20000000，Lv5 100000000。
    今日已领取:
      check:
        - 成功领取今日额度后设为 true。
        - 新一天或系统刷新后设为 false。
    商城权限:
      check:
        - 根据等级或任务解锁更新：Lv1 金钱道具，Lv2 感官强化，Lv3 心理干涉，Lv4 常识修改，Lv5 现实权限。
        - 这是 EJS 会读取的控制变量，必须保留。
    风险等级:
      check:
        - 根据失败报价、目标警觉、心理干涉、常识补丁、曝光风险更新。
        - 可取 低、中、高、极高。
        - 这是 EJS 会读取的控制变量，必须保留。

  界面状态:
    当前聚焦目标:
      check:
        - 玩家选择、观察、报价或行动指向某目标时写入目标名称。
        - 无明确目标时可保持原值或置空。
    当前目标状态:
      check:
        - 应与当前聚焦目标在 `目标库` 中的 `当前状态` 同步。
        - 没有建档时使用 未接触。
    当前行动类型:
      check:
        - 根据本轮玩家行动写入：未指定、原味交易、私密素材、线上支配、线下见面、恶堕转折、资产化维护、购买道具、常识补丁。
    当前场景:
      check:
        - 写入本轮主要场景，如 私信、直播后台、酒店、会所、车内、公寓、办公室。
    推荐加载阶段:
      check:
        - 默认 自动。
        - 若本轮需要强制加载某阶段，可写入 原味交易、私密素材、线上支配、线下见面、恶堕转折、资产化。

  目标库:
    check:
      - 新增单个目标时可直接 insert 到 `目标库.目标名`；新增多个目标时可 replace 整个 `目标库`，保留已有目标，并加入新目标完整档案。
      - 目标必须是成年虚构角色；年龄不得低于 18。
      - 保留字段只有：身份、年龄、估值金额、最低试探报价、当前状态、当前突破口、贪欲、自尊、羞耻、堕落度、警觉度、最近交易。
      - 不要写 职业、社会价值等级、认知稳定、认知日志。
      - 当前状态按事件推进：未接触 -> 已试探 -> 已交易 -> 已屈服 -> 已资产化。
      - 数值单次变化通常控制在 1-15；重大交易、心理干涉或常识补丁可以更大，但要在剧情里写出原因。
      - 最近交易用于压缩记录最新关键事件，包括价格、结果、条件和事后变化。
      - 若更新当前聚焦目标，同步更新 `界面状态.当前目标状态`。

  商城:
    已购买道具:
      check:
        - 只记录已购买道具，不记录完整库存。
        - 商品价格、描述、解锁等级由世界书和前端商品表提供。
        - 购买新道具时可直接 insert 到 `商城.已购买道具.道具名`；批量整理时可 replace 整个 `商城.已购买道具`。
        - 道具记录格式为 `{数量: 1, 层级: 金钱道具}`。
        - 已有道具再次购买可对数量 delta +1。
        - 消耗道具成功使用后可 delta -1；数量为 0 时 replace 父对象移除。

  任务:
    check:
      - 任务保留，用于推动系统升级和玩法目标。
      - 新增单个任务时可直接 insert 到 `任务.任务名`；新增多个任务时可 replace 整个 `任务`，保留已有任务，并加入新任务。
      - 任务字段只保留：类型、要求、进度、奖励资金、推荐目标、状态。
      - 不再记录奖励经验。
      - 任务完成时可结算奖励资金，并更新系统等级或商城权限。

  $flag:
    首次报价:
      check:
        - 第一笔非公开报价被目标明确回应后设为 true。
    首次购买道具:
      check:
        - 第一件商城道具购买成功后设为 true。
    首次常识补丁:
      check:
        - 第一次常识补丁成功在剧情中生效后设为 true。
        - 不需要写入常识规则表，只需更新 flag、风险等级、最近交易或任务。
</mvu_update_rules>
</money_mvu_rules>
